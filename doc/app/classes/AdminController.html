Class AdminController
In: app/controllers/admin_controller.rb
Parent: ApplicationController

Controls various administrative tasks. These actions include translating texts, creating other admins, and editing tests. Many of the other admin tasks are kept in their respective controllers, i.e. editing / viewing user details is in the user_controller

This was done to ultimately make the code more intuitive to follow, but isn‘t 100% consistent. Hence the use of this controller to take care of loose ends and actions which don‘t seem to fit specifically in their own controller.

new_forum()

Creates a new forum, based on the forum_type parameter fed to it. This allows for easy creation of the two different types of forums. The only time this is needed is when you‘ve created a semester from scratch. Then the veiw for index will display links to create the new forums. If a forum is copied from a previous one then new forums are automatically created in the semester_controller
